    Installation Guide¶

    This guide covers installing HuskyCats both locally and in GitLab projects.

    Prerequisites¶

    • Docker or Podman: The only requirement (all other tools are containerized)
    • Git Repository: The project must be a Git repository
    • Docker Compose or Podman Compose: Usually included with Docker/Podman

    ðŸš€ Quick Start - Architecture-Aware Installation¶

    HuskyCats now automatically detects your system architecture and uses the appropriate Docker image.

    The Universal One-Liner¶

    # This command auto-detects your architecture (arm64 or amd64) and uses SSH by default
curl -fsSL https://gitlab.com/jsullivan2_bates/pubcontainers/-/raw/main/install.sh | bash

# To use HTTPS instead of SSH
curl -fsSL https://gitlab.com/jsullivan2_bates/pubcontainers/-/raw/main/install.sh | bash -s -- --no-ssh

    Check Your Architecture First¶

    # Find out your system architecture
uname -m
# x86_64 = amd64 (most servers, CI runners)
# arm64/aarch64 = ARM (Apple Silicon Macs, ARM servers)

    Local Development Installation¶

    If you're working with the HuskyCats repository itself:

    1. Clone and Build¶

    # Clone the repository
git clone https://gitlab.com/jsullivan2_bates/pubcontainers.git
cd pubcontainers

# Build the Docker image
docker build -t husky-lint:local .

# Verify the build
docker images | grep husky-lint

    2. Test Locally¶

    # Test the linters on the HuskyCats codebase itself
docker-compose run --rm lint

# Run specific linters
docker-compose run --rm shellcheck
docker-compose run --rm hadolint

    Installing in Other Projects¶

    Method 1: One-Line Install Script (Recommended)¶

    The install script automatically detects your architecture and uses the correct Docker platform:

    # Navigate to your project
cd /path/to/your/project

# Ensure SSH access to GitLab (for SSH authentication)
ssh -T git@gitlab.com

# Run the universal installer (uses SSH by default)
curl -fsSL https://gitlab.com/jsullivan2_bates/pubcontainers/-/raw/main/install.sh | bash

# Or with wget
wget -qO- https://gitlab.com/jsullivan2_bates/pubcontainers/-/raw/main/install.sh | bash

# To use HTTPS instead
curl -fsSL https://gitlab.com/jsullivan2_bates/pubcontainers/-/raw/main/install.sh | bash -s -- --no-ssh

    Method 2: Direct Docker Commands (Architecture-Specific)¶

    If you prefer to run Docker directly or need to specify the architecture:

    # For x86_64/amd64 systems (most servers, GitLab CI):
docker run --rm --platform linux/amd64 -v "$(pwd):/workspace" \
  registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

# For ARM64 systems (Apple Silicon Macs):
docker run --rm --platform linux/arm64 -v "$(pwd):/workspace" \
  registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

# Let Docker auto-detect (may fail if architecture mismatch):
docker run --rm -v "$(pwd):/workspace" \
  registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

    This will: - Create all configuration files - Set up Git hooks - Install GitLab CI templates - Configure npm scripts

    Method 3: Using Docker Compose¶

    If you want to copy the docker-compose.yml to your project:

    # Copy docker-compose.yml from HuskyCats
cp /path/to/huskycats/docker-compose.yml docker-compose.husky.yml

# Update the image reference
sed -i 's/husky-lint:local/registry.gitlab.com\/jsullivan2_bates\/pubcontainers\/husky-lint:latest/g' docker-compose.husky.yml

# Run linters
docker-compose -f docker-compose.husky.yml run --rm lint

    Post-Installation¶

    Verify Installation¶

    # Test the setup
docker-compose run --rm lint

# Check Git hooks
git add . && git commit -m "test: verify hooks"

# List installed configurations
ls -la .lintstagedrc.json pyproject.toml .flake8 .ansible-lint

# Check GitLab CI template
cat .gitlab/ci/husky-lint.yml

    Using the Tools¶

    After installation, you can run linters using:

    # Using npm scripts (runs in container)
npm run lint      # Run all linters
npm run lint:fix  # Auto-fix issues

# Using docker-compose
docker-compose run --rm lint
docker-compose run --rm lint-fix
docker-compose run --rm shell

# Using Docker directly
docker run --rm -v "$(pwd):/workspace" husky-lint:local npm run lint
    1. Check Husky is working: 
      git add .
git commit -m "test: verify husky setup"

    You should see the pre-commit hook running.

    1. Test linting: 
      ./scripts/comprehensive-lint.sh --all --verbose

    Configure Your IDE¶

    VS Code¶

    Add to .vscode/settings.json: 

    {
  "editor.formatOnSave": true,
  "python.formatting.provider": "black",
  "python.linting.enabled": true,
  "python.linting.flake8Enabled": true,
  "[python]": {
    "editor.codeActionsOnSave": {
      "source.organizeImports": true
    }
  }
}

    PyCharm¶

    1. Go to Settings â†’ Tools â†’ File Watchers
    2. Add watchers for Black and flake8
    3. Enable "Reformat on Save"

    Update package.json Scripts¶

    Add these helpful scripts: 

    {
  "scripts": {
    "lint": "./scripts/comprehensive-lint.sh --all",
    "lint:fix": "./scripts/comprehensive-lint.sh --all --fix",
    "lint:staged": "./scripts/comprehensive-lint.sh --staged",
    "prepare": "husky install"
  }
}

    Troubleshooting¶

    Architecture Issues¶

    1. "exec format error" when running Docker 

      # This means architecture mismatch. Use the platform flag:
docker run --rm --platform linux/amd64 -v "$(pwd):/workspace" \
  registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

# Or use the install script which auto-detects:
curl -fsSL https://gitlab.com/jsullivan2_bates/pubcontainers/-/raw/main/install.sh | bash

    2. Check your system architecture 

      # See what architecture you're running
uname -m
# x86_64 = use linux/amd64
# arm64 or aarch64 = use linux/arm64

# Check Docker image architecture
docker inspect registry.gitlab.com/bates-ils/projects/trustees-portal/sid-controller/huskycats-bates/husky-lint:latest | grep Architecture

    Common Issues¶

    1. "command not found" errors
    2. Ensure tools are installed and in PATH

    3. Try using full paths in scripts

    4. Permission denied

    5. Make scripts executable: chmod +x <script>

    6. Check file permissions

    7. Python tools not found

    8. Activate virtual environment

    9. Install missing tools with pip

    10. Husky hooks not running

    11. Run npx husky install
    12. Check Git version (needs 2.9+)
    13. Ensure core.hooksPath is not set

    See Troubleshooting Guide for more solutions.
