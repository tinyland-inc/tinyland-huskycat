    Claude Desktop Integration Guide¶

    This guide covers integrating HuskyCats MCP Server with Claude Desktop for AI-powered code validation.

    Table of Contents¶

    1. Quick Start
    2. Deployment Options
    3. Configuration
    4. Using with Claude
    5. Troubleshooting

    Quick Start¶

    Option 1: NPM Package (Easiest)¶

    # Install globally
npm install -g @huskycats/mcp-server

# Add to Claude Desktop config
cat >> ~/.claude/mcp.json << EOF
{
  "mcpServers": {
    "huskycats": {
      "command": "huskycats-mcp",
      "args": ["--stdio"],
      "type": "stdio"
    }
  }
}
EOF

# Restart Claude Desktop

    Option 2: Local Development¶

    # Clone and run
git clone https://github.com/huskycats/huskycats-bates.git
cd huskycats-bates/mcp-server
npm install
npm start

# Configure Claude (see below)

    Option 3: Podman Container¶

    # Deploy with Podman
cd mcp-server
./deploy-podman.sh
# Choose: [1] Alpine (lightweight) or [2] Rocky (full features)

# Configure Claude for HTTP endpoint

    Deployment Options¶

    1. Stdio Mode (Recommended for Personal Use)¶

    Direct process communication. No network setup required.

    Pros: - Simple setup - No authentication needed - Works offline

    Cons: - Single user only - Requires local installation

    Configuration: 

    {
  "mcpServers": {
    "huskycats": {
      "command": "npx",
      "args": ["@huskycats/mcp-server", "--stdio"],
      "type": "stdio"
    }
  }
}

    2. HTTP Mode (Recommended for Teams)¶

    Network-based communication. Supports remote servers.

    Pros: - Multi-user support - Remote access - Centralized management

    Cons: - Requires authentication - Network setup needed

    Configuration: 

    {
  "mcpServers": {
    "huskycats": {
      "url": "http://localhost:8080/rpc",
      "type": "http",
      "authentication": {
        "type": "bearer",
        "token": "your-secret-token"
      }
    }
  }
}

    3. Podman Desktop Integration¶

    Full containerized environment with GUI.

    Pros: - Complete isolation - Easy management - Visual monitoring

    Cons: - Requires Podman Desktop - More resource usage

    Setup: 

    # 1. Install Podman Desktop
# Download from: https://podman-desktop.io/

# 2. Deploy HuskyCats
cd mcp-server
./deploy-podman.sh

# 3. Get configuration
# The script outputs Claude configuration

    Configuration¶

    Basic Configuration¶

    Location: ~/.claude/mcp.json (or %APPDATA%\Claude\mcp.json on Windows)

    {
  "mcpServers": {
    "huskycats": {
      "url": "http://localhost:8080/rpc",
      "type": "http",
      "description": "Code validation and linting tools",
      "authentication": {
        "type": "bearer",
        "token": "${HUSKYCATS_TOKEN}"
      }
    }
  }
}

    Advanced Configuration¶

    With all features enabled:

    {
  "mcpServers": {
    "huskycats": {
      "url": "http://localhost:8080/rpc",
      "type": "http",
      "description": "Code validation with file system access",
      "capabilities": {
        "tools": true,
        "resources": true,
        "prompts": true,
        "notifications": true
      },
      "authentication": {
        "type": "bearer",
        "token": "${HUSKYCATS_TOKEN}"
      },
      "features": {
        "syncthing": {
          "enabled": true,
          "deviceId": "${SYNCTHING_DEVICE_ID}"
        },
        "hooks": {
          "enabled": true,
          "preOperation": "npx @huskycats/hooks pre",
          "postOperation": "npx @huskycats/hooks post"
        }
      },
      "metadata": {
        "version": "2.0.0",
        "protocolVersion": "2024-11-05"
      }
    }
  }
}

    Environment Variables¶

    Set these before starting Claude Desktop:

    # Required for HTTP mode
export HUSKYCATS_TOKEN="your-secret-token"

# Optional for Syncthing
export SYNCTHING_DEVICE_ID="your-device-id"

# Optional for custom server
export HUSKYCATS_URL="https://validator.company.com/rpc"

    Using with Claude¶

    Available Commands¶

    Once configured, you can ask Claude to:

    Validation: - "Validate my Python code" - "Check this file for linting errors" - "Run flake8 on all Python files"

    Formatting: - "Format this code with Black" - "Fix the formatting issues" - "Make this code PEP-8 compliant"

    Security: - "Check for security issues" - "Run bandit on my Python files" - "Find potential vulnerabilities"

    Infrastructure: - "Validate my Dockerfile" - "Check my shell scripts" - "Lint my YAML files"

    Example Conversations¶

    Basic Validation: 

    You: Check my Python file app.py for issues

Claude: I'll validate app.py using the HuskyCats tools.

[Uses python-flake8 tool]

I found the following issues:
- Line 5: Unused import 'sys'
- Line 12: Missing whitespace around operator
- Line 20: Line too long (95 > 79 characters)

Would you like me to fix these issues?

    Full Project Validation: 

    You: Run a complete validation on my project

Claude: I'll run a comprehensive validation suite on your project.

[Uses full_validation prompt]

Validation Results:
✅ Python: 12 files checked, 3 issues found
✅ JavaScript: 8 files checked, all clean
⚠️ Shell: 2 files checked, 1 issue found
✅ Docker: 1 file checked, all clean

Would you like me to show the detailed issues or fix them automatically?

    Working with Resources¶

    If Resources API is enabled, Claude can access your files:

    You: Review my project structure and suggest improvements

Claude: I'll examine your project structure using the file system resources.

[Uses resources/list and resources/read]

Here's what I found:
- Missing __init__.py in src/utils/
- Inconsistent naming: mixing camelCase and snake_case
- No .gitignore file
- Missing type hints in Python files

Would you like me to help fix these issues?

    Troubleshooting¶

    Common Issues¶

    1. "MCP Server not found" - Check Claude Desktop is restarted after config changes - Verify mcp.json syntax is valid JSON - Ensure server is running (for HTTP mode)

    2. "Authentication failed" - Check token matches between server and config - Ensure token is properly quoted in JSON - Try regenerating token

    3. "Tools not available" - Verify server is initialized properly - Check server logs for errors - Ensure capabilities include "tools": true

    4. "Cannot access files" - Resources API requires proper permissions - Check volume mounts for containers - Verify file paths are absolute

    Debug Mode¶

    Enable debug logging:

    {
  "mcpServers": {
    "huskycats": {
      "url": "http://localhost:8080/rpc",
      "type": "http",
      "debug": true,
      "logLevel": "debug"
    }
  }
}

    Server Logs¶

    Check server logs for issues:

    # For npm/development
npm run dev

# For container
podman logs huskycats-mcp-server

# For systemd
journalctl -u huskycats-mcp

    Testing Connection¶

    Test the server is accessible:

    # Health check
curl http://localhost:8080/health

# List tools
curl http://localhost:8080/tools

# Test RPC endpoint
curl -X POST http://localhost:8080/rpc \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-token" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/list",
    "id": 1
  }'

    Security Considerations¶

    For Personal Use¶

    • Use stdio mode to avoid network exposure
    • Keep tokens in environment variables
    • Don't commit mcp.json with tokens

    For Team Use¶

    • Use HTTPS in production
    • Rotate tokens regularly
    • Implement IP allowlisting
    • Monitor access logs

    For Public Deployment¶

    • Enable all security features (fail2ban, firewalld)
    • Use Kubernetes with proper RBAC
    • Implement rate limiting
    • Regular security audits
