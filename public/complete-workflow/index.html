    HuskyCats Complete Workflow Guide¶

    This guide demonstrates the complete validation workflow using HuskyCats with git hooks and the MCP server for AI-assisted fixes.

    Architecture Overview¶

    1. Local Container - Embedded validation tools (Black, Flake8, ESLint, etc.)
    2. Git Hooks - Pre-commit and pre-push validation
    3. HTTP MCP Server - AI assistance for fixing validation issues
    4. Syncthing - File synchronization for distributed validation

    Setup Process¶

    1. Install Git Hooks¶

    # Copy hooks from husky-hooks directory
cp -r husky-hooks/* .git/hooks/
chmod +x .git/hooks/pre-commit .git/hooks/pre-push .git/hooks/commit-msg

    2. Run MCP Server¶

    The MCP server provides HTTP endpoints for validation tools:

    # Using existing container (already running on port 8080)
podman ps | grep mcp-server

# Or start fresh
cd mcp-server
podman-compose up -d

    3. Verify MCP Server¶

    # Check health
curl http://localhost:8080/health

# Initialize session
curl -X POST http://localhost:8080/rpc \
  -H "Content-Type: application/json" \
  -H "X-Session-ID: my-session" \
  -d '{
    "jsonrpc": "2.0",
    "method": "initialize",
    "params": {"protocolVersion": "2024-11-05", "capabilities": {}},
    "id": 1
  }'

# List available tools
curl -X POST http://localhost:8080/rpc \
  -H "Content-Type: application/json" \
  -H "X-Session-ID: my-session" \
  -d '{"jsonrpc": "2.0", "method": "tools/list", "params": {}, "id": 2}'

    Validation Workflow¶

    Step 1: Make Changes with Issues¶

    Create a file with common issues:

    # test_bad_code.py
import os
import  json # extra space
def badFunction( ):  # bad formatting
    x=1  # no spaces
    y =2  # unused variable
    return x

    Step 2: Attempt Commit (Fails)¶

    git add test_bad_code.py
git commit -m "test: add bad code"

# Pre-commit hook runs validation:
# ✅ Black formats the code
# ❌ Flake8 finds issues
# ❌ Commit blocked

    Step 3: Use MCP Server to Fix¶

    The MCP server provides HTTP API for validation tools:

    # Format with Black
curl -X POST http://localhost:8080/rpc \
  -H "Content-Type: application/json" \
  -H "X-Session-ID: my-session" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": {
      "name": "python-black",
      "arguments": {"files": ["test_bad_code.py"], "fix": true}
    },
    "id": 3
  }'

# Check style with Flake8
curl -X POST http://localhost:8080/rpc \
  -H "Content-Type: application/json" \
  -H "X-Session-ID: my-session" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": {
      "name": "python-flake8",
      "arguments": {"files": ["test_bad_code.py"]}
    },
    "id": 4
  }'

    Step 4: Fix Remaining Issues¶

    Based on validation output: - Remove unused imports - Fix variable usage - Address security issues

    Step 5: Successful Commit¶

    git add test_bad_code.py
git commit -m "test: add properly formatted code"
# ✅ All validations pass
# ✅ Commit succeeds

    Available Validation Tools¶

    The MCP server exposes these tools via HTTP:

    Python¶

    • python-black - Code formatter
    • python-flake8 - Style checker
    • python-mypy - Type checker
    • python-bandit - Security scanner

    JavaScript/TypeScript¶

    • js-eslint - Linter
    • js-prettier - Formatter

    Infrastructure¶

    • shell-shellcheck - Shell script linter
    • docker-hadolint - Dockerfile linter
    • yaml-yamllint - YAML validator
    • gitlab-ci-validate - GitLab CI validator

    Other¶

    • ansible-lint - Ansible playbook linter

    Integration with AI Assistants¶

    The MCP server is designed for AI assistants like Claude to help fix validation issues:

    1. Session Management - Each AI session gets a unique ID
    2. Tool Discovery - AI can list available tools
    3. Automated Fixes - AI can run formatters with fix: true
    4. Error Analysis - AI receives detailed validation output

    Pre-Push CI Validation¶

    The pre-push hook runs additional CI checks:

    # Runs automatically on git push
- GitLab CI schema validation
- Security scanning
- Dependency checks
- Container build verification

    Troubleshooting¶

    MCP Server Not Responding¶

    # Check if running
podman ps | grep mcp-server

# View logs
podman logs mcp-server

# Restart
podman restart mcp-server

    Validation Errors¶

    # Run validation manually
podman run --rm -v $(pwd):/workspace:z localhost/huskycat:latest black --check test.py
podman run --rm -v $(pwd):/workspace:z localhost/huskycat:latest flake8 test.py

    Hook Issues¶

    # Bypass hooks temporarily
git commit --no-verify -m "emergency fix"

# Re-enable by ensuring hooks are executable
chmod +x .git/hooks/*

    Best Practices¶

    1. Always run validation before pushing - Catches issues early
    2. Use MCP server for complex fixes - AI can help resolve tricky issues
    3. Keep tools updated - Rebuild container periodically
    4. Customize rules - Edit .flake8, .prettierrc, etc. for project needs
    5. Document exceptions - Use inline comments for necessary violations

    Summary¶

    This workflow ensures: - ✅ Code quality through automated validation - ✅ Consistent formatting across the team - ✅ Security issues caught before deployment - ✅ AI assistance for fixing complex issues - ✅ CI/CD pipeline success through pre-push validation
