    One-Line Installation Guide¶

    This guide explains how to install HuskyCats using our containerized one-line installation.

    Prerequisites¶

    • Docker: The only requirement (all tools are containerized)
    • Git repository: Your project must be initialized as a Git repository

    Local Development¶

    If you're working with HuskyCats locally:

    # Build the image
docker build -t husky-lint:local .

# Install in current directory
docker run --rm -v "$(pwd):/workspace" husky-lint:local

# Use the tools
docker-compose run --rm lint

    Production Installation¶

    Using the published GitLab registry image:

    # One-line install (after image is published to GitLab)
docker run --rm -v "$(pwd):/workspace" registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

    What Happens During Installation¶

    The one-line command will: 1. Pull the container with all tools pre-installed 2. Mount your current directory as a workspace 3. Create all configuration files (.flake8, pyproject.toml, etc.) 4. Set up Git hooks (.husky/pre-commit, .husky/commit-msg) 5. Install GitLab CI/CD templates 6. Configure npm scripts for easy usage

    Using a Specific Version¶

    docker run --rm -v "$(pwd):/workspace" -w /workspace registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:v1.0.0

    With GitLab Authentication¶

    If your GitLab registry requires authentication:

    # Login to GitLab registry first
docker login registry.gitlab.com

# Then run the installation
docker run --rm -v "$(pwd):/workspace" -w /workspace registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

    Method 2: Direct Script Installation¶

    The script now uses SSH authentication by default for better security with private container registries.

    Default: SSH Authentication (Recommended)¶

    # Ensure SSH access to GitLab first
ssh -T git@gitlab.com

# Run installer (uses SSH by default)
curl -fsSL https://gitlab.com/jsullivan2_bates/pubcontainers/-/raw/main/install.sh | bash

    Using HTTPS Mode¶

    # Use --no-ssh flag to disable SSH authentication
curl -fsSL https://gitlab.com/jsullivan2_bates/pubcontainers/-/raw/main/install.sh | bash -s -- --no-ssh

    With GitLab Token¶

    # Set GITLAB_TOKEN for private registry access
GITLAB_TOKEN="your-personal-access-token" \
curl -fsSL https://gitlab.com/jsullivan2_bates/pubcontainers/-/raw/main/install.sh | bash

    With Custom GitLab Instance¶

    GITLAB_URL=https://gitlab.company.com \
GITLAB_PROJECT=group/huskycats \
curl -fsSL https://gitlab.company.com/group/huskycats/-/raw/main/install.sh | bash

    What Gets Installed¶

    The one-line installer sets up:

    1. Git Hooks
    2. pre-commit: Runs linters on staged files
    3. commit-msg: Validates commit message format
    4. pre-push: Runs full validation before push

    5. post-merge: Updates dependencies after merge

    6. Linting Tools

    7. Python: Black, flake8, mypy, ruff, pylint, isort, bandit
    8. JavaScript/TypeScript: ESLint, Prettier
    9. Ansible: ansible-lint
    10. Shell: shellcheck
    11. Docker: hadolint

    12. And more...

    13. Configuration Files

    14. .lintstagedrc.json: File-specific linting rules
    15. pyproject.toml: Python tools configuration
    16. .flake8: Python linting rules
    17. .prettierrc: Code formatting rules

    18. .gitlab-ci.yml: CI/CD pipeline configuration

    19. Scripts

    20. scripts/comprehensive-lint.sh: Smart linting script

    21. scripts/enhanced-ci-validation.sh: CI validation

    22. GitLab CI/CD

    23. .gitlab/ci/husky-lint.yml: Comprehensive CI/CD template
    24. Automatic inclusion in .gitlab-ci.yml

    Post-Installation Steps¶

    After installation completes:

    1. Review Changes 

      git status
git diff

    2. Commit the Setup 

      git add .
git commit -m "feat: add comprehensive linting and CI/CD setup"

    3. Push to GitLab 

      git push origin main

    4. Test the Setup 

      # Run all linters
npm run lint

# Auto-fix issues
npm run lint:fix

# Test pre-commit hook
echo "test" > test.py
git add test.py
git commit -m "test: verify linting"

    Customization¶

    Environment Variables¶

    You can customize the installation with environment variables:

    # Skip Python virtual environment creation
SKIP_VENV=true docker run --rm -v "$(pwd):/workspace" -w /workspace registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

# Use specific Node package manager
PKG_MANAGER=yarn docker run --rm -v "$(pwd):/workspace" -w /workspace registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

    Selective Installation¶

    To install only specific components:

    # Only Git hooks and Node.js linting
SKIP_PYTHON=true docker run --rm -v "$(pwd):/workspace" -w /workspace registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

# Only Python linting
SKIP_NODE=true docker run --rm -v "$(pwd):/workspace" -w /workspace registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

    Updating¶

    To update to the latest version:

    # Pull latest image
docker pull registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

# Re-run installation
docker run --rm -v "$(pwd):/workspace" -w /workspace registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

    Uninstalling¶

    To remove the setup:

    # Remove Git hooks
rm -rf .husky

# Remove configuration files
rm -f .lintstagedrc.json pyproject.toml .flake8 .prettierrc .prettierignore

# Remove scripts
rm -rf scripts

# Remove from package.json
npm uninstall husky lint-staged

# Remove GitLab CI templates
rm -rf .gitlab/ci

    Troubleshooting¶

    Docker Permission Issues¶

    If you encounter permission issues with Docker:

    # Run with user mapping
docker run --rm -v "$(pwd):/workspace" -w /workspace -u $(id -u):$(id -g) registry.gitlab.com/jsullivan2_bates/pubcontainers/husky-lint:latest

    Network Issues¶

    For environments behind proxies:

    # With HTTP proxy
docker run --rm \
  -v "$(pwd):/workspace" \
  -w /workspace \
  -e HTTP_PROXY=$HTTP_PROXY \
  -e HTTPS_PROXY=$HTTPS_PROXY \
  registry.gitlab.com/bates-ils/projects/trustees-portal/sid-controller/huskycats-bates/husky-lint:latest
